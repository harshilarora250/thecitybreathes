## Overview

The City Breathes turns live city data (traffic, air quality, weather, and social activity) into an embodied, interactive 3D artwork. Instead of showing data as numbers or charts, it lets the city become a living organism: a digital sculpture that breathes, changes color and shape, and вЂњsingsвЂќ through procedural sound.

This is not a dashboard. It is a new artistic medium that is:

- **Interactive** вЂ“ users can touch and influence the sculpture.
- **Data-driven** вЂ“ live signals shape motion, geometry, color, and sound.
- **Procedural** вЂ“ every moment is unique, generated by algorithms.
- **Evolving** вЂ“ the artwork continuously changes over time.
- **Networked** вЂ“ it depends on real urban systems to exist.

## Features

- Living 3D icohedron sculpture that breathes and deforms based on city data.
- Real-time color, motion, and particle effects driven by traffic, air quality, weather, and social activity.
- Interactive breathing styles: users can click/tap the sculpture to change its behavior.
- Procedural audio generated in the browser using the Web Audio API.
- City selection with fallback/demo mode when live APIs are unavailable.
- Time-lapse / вЂњpast dayвЂќ visualization mode.

## API & Data Sources

- **Weather & Air Quality**: [Open-Meteo](https://open-meteo.com/) вЂ” live, keyless, CORS-friendly (temperature, humidity, wind, cloud, precipitation; PM2.5/PM10/NOв‚‚/Oв‚ѓ).
- **Traffic**: [TomTom Traffic API](https://developer.tomtom.com/) вЂ” real congestion, fetched via the server proxy `GET /api/traffic` (`TOMTOM_TRAFFIC_KEY`). Falls back to a time-of-day estimate when the key is absent.
- **Social Activity**: Procedural generation simulating public activity patterns.

## Demo Mode

When live APIs are unavailable, the application automatically uses procedurally generated fallback data based on city coordinates and time of day. This ensures the sculpture remains functional during presentations and judging.
